# 文档目录

这个目录用来告诉你：每份文档给谁看、什么时候看。

## 1. 你自己先看

| 文档 | 作用 |
| --- | --- |
| `小白上手指南.md` | 解释项目是什么、怎么看代码、怎么启动 |
| `后端学习路线.md` | 按 Java 后端学习顺序拆解项目 |
| `数据库表结构说明.md` | 解释每张表是干什么的 |

## 2. 给前端同学看

| 文档 | 作用 |
| --- | --- |
| `前端对接.md` | 前端怎么调接口、怎么带 token、怎么展示返回数据 |
| `接口契约文档.md` | 完整接口字段、枚举、请求和响应示例 |
| `Figma设计交接稿.md` | 页面结构、星空图、涂鸦页和后台页的设计建议 |
| `发给前端同学的消息.md` | 可以直接发给前端同学的说明文字 |

## 3. 给 AI 同学看

| 文档 | 作用 |
| --- | --- |
| `AI-RAG对接说明.md` | 说明 RAG 怎么接、后端会给 AI 哪些上下文 |
| `接口契约文档.md` | 查看 AI 返回字段和主流程接口 |

## 4. 给整个小组看

| 文档 | 作用 |
| --- | --- |
| `团队协作说明.md` | 三个人怎么分工、怎么提任务、怎么联调 |
| `发给三人小群的消息.md` | 可以直接发到小群里的项目交接消息 |
| `GitHub首次发布说明.md` | 第一次把项目推到 GitHub 的步骤 |

## 5. 最推荐的阅读顺序

如果你现在刚打开项目，按这个顺序看：

```text
1. README.md
2. docs/文档目录.md
3. docs/小白上手指南.md
4. docs/接口契约文档.md
5. docs/前端对接.md
6. docs/AI-RAG对接说明.md
```
