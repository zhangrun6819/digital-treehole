# 数字树洞 Figma 设计交接稿

这份文档是给前端同学和做设计的人看的。

你这边当前 Figma 账号权限是 `View`，所以我暂时不能直接帮你新建或改 Figma 文件。先把设计骨架定出来，前端同学拿着这份就能去画，不会乱。

## 1. 这次 Figma 里建议建哪些页面

### 页面 1：封面与版本记录

- 文件名建议：`Digital Treehole`
- 内容：
  - 项目名
  - 小组成员
  - 当前版本
  - 最近更新时间

### 页面 2：用户端页面

建议至少放 5 个主界面：

- 首页 / 进入页
- 创建或进入聊天会话页
- 聊天页
- 涂鸦输入弹层或页面
- 情感星空图页

### 页面 3：管理员页面

建议至少放 3 个页面：

- 管理员登录页
- 风险事件列表页
- 规则 / 援助资源维护页

### 页面 4：设计规范

- 主色
- 辅助色
- 字体层级
- 按钮样式
- 输入框样式
- 卡片样式
- 状态色

## 2. 页面结构建议

### 用户端首页

建议放这些元素：

- 项目标题
- 一句温和的引导文案
- “开始倾诉”按钮
- “看看我的情感星空”入口
- 底部的隐私说明

页面气质关键词：

- 温柔
- 安静
- 有陪伴感
- 不要像后台管理系统

### 聊天页

建议结构：

- 顶部：会话标题 + 陪伴风格
- 中间：消息列表
- 底部：文本输入框 + 发送按钮 + 涂鸦入口
- 侧边或底部：需要时展示心理援助资源

消息视觉建议：

- 用户消息和 AI 消息要区分颜色和对齐方式
- AI 回复里如果有 `followUpQuestion`，可以单独弱化展示

### 涂鸦输入

建议交互：

- 弹出画板
- 画完后选择一个情绪标签
- 可写一句短说明
- 提交后回到聊天流里展示

## 3. 情感星空图怎么做才像作品

这个页面是比赛里很容易出彩的地方。

建议前端在 Figma 里先画一个视觉方向，不要只放普通柱状图。

建议包含：

- 深色或柔和渐变背景
- 星点分布区
- 当前主情绪展示
- 最近 7 天总结文案
- 图例说明

和后端接口对应的字段是：

- `points[].x`
- `points[].y`
- `points[].radius`
- `points[].color`
- `points[].emotionTag`
- `points[].label`
- `dominantEmotion`
- `summaryText`

## 4. 陪伴风格在视觉上怎么区分

项目里现在有 3 种风格：

- `GENTLE_ELDER`
- `PEER`
- `HUMOROUS`

Figma 里建议至少在这两处体现差别：

- 会话创建页的风格选择卡片
- 聊天页顶部的当前风格标识

## 5. 设计不要漏掉的文案

这些文案最好在界面里提前留位置：

- 匿名说明
- 数据仅用于本次体验的提示
- 高风险时的援助提示
- 涂鸦上传限制说明

## 6. Figma 交接给后端时最有用的东西

如果前端同学想让后端更好配合，最好给这些：

- 页面截图
- 每个页面对应什么接口
- 特殊状态怎么展示
- 哪些字段是必需，哪些是可选

## 7. 直接可用的页面清单

可以把下面这串直接发给前端同学，让他按这个顺序画：

```text
2. 聊天会话列表页
3. 聊天页
4. 涂鸦弹层
5. 情感星空图页
6. 管理员登录页
7. 风险事件列表页
8. 规则维护页
```
