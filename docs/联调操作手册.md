# 联调操作手册

这份文档给后端、前端、AI 同学一起用。目标是把“怎么联调”变成可照着执行的步骤。

## 1. 第一轮联调约定

| 项目 | 约定 |
| --- | --- |
| 联调方式 | 本机或同一局域网 |
| 后端地址 | `http://<后端电脑局域网IP>:8080` |
| AI 模式 | 后端默认 `mock` |
| 公网部署 | 暂不作为第一轮目标 |
| 接口依据 | `docs/接口契约文档.md` 和 Swagger |

如果前端和后端在同一台电脑，用：

```text
http://localhost:8080
```

如果前端在另一台电脑，必须用后端电脑的局域网 IP，不能用前端自己的 `localhost`。

## 2. 后端联调步骤

进入项目目录：

```powershell
cd path\to\digital-treehole
```

先跑测试：

```powershell
.\scripts\run-test.ps1
```

成功信号：

```text
BUILD SUCCESS
```

启动后端：

```powershell
.\scripts\run-dev.ps1
```

如果 MySQL root 有密码：

```powershell
.\scripts\run-dev.ps1 -DbPassword "你的MySQL密码"
```

打开 Swagger：

```text
http://localhost:8080/swagger-ui.html
```

查看局域网地址：

```powershell
.\scripts\show-lan-ip.ps1
```

把脚本输出的地址发给前端同学：

```text
http://<后端电脑局域网IP>:8080
http://<后端电脑局域网IP>:8080/swagger-ui.html
```

## 3. 前端联调步骤

前端先统一配置后端地址：

```js
const API_BASE_URL = 'http://<后端电脑局域网IP>:8080'
```

如果前端和后端在同一台电脑，可以临时用：

```js
const API_BASE_URL = 'http://localhost:8080'
```

### 用户主流程

第一步，创建匿名身份：

```http
POST /api/v1/visitors/session
```

把返回的 `data.token` 存起来：

```js
localStorage.setItem('treehole_token', data.token)
```

后续用户侧接口都带：

```http
Authorization: Bearer <token>
```

第二步，创建会话：

```http
POST /api/v1/chat/sessions
Content-Type: application/json
Authorization: Bearer <token>
```

```json
{
  "companionStyle": "PEER",
  "title": "考试压力"
}
```

第三步，发送文本消息：

```http
POST /api/v1/chat/sessions/{sessionId}/messages
Content-Type: application/json
Authorization: Bearer <token>
```

```json
{
  "inputType": "TEXT",
  "content": "我最近因为考试很焦虑，但还是想试着说出来"
}
```

第四步，查询消息列表：

```http
GET /api/v1/chat/sessions/{sessionId}/messages?pageNo=1&pageSize=20
Authorization: Bearer <token>
```

第五步，查询情感星空图：

```http
GET /api/v1/chat/stats/star-map?days=7
Authorization: Bearer <token>
```

### 涂鸦流程

先上传 PNG：

```http
POST /api/v1/doodles
Content-Type: multipart/form-data
Authorization: Bearer <token>
```

字段名：

```text
file
```

再发送涂鸦消息：

```http
POST /api/v1/chat/sessions/{sessionId}/messages
Content-Type: application/json
Authorization: Bearer <token>
```

```json
{
  "inputType": "DOODLE",
  "doodleAssetId": 1,
  "emotionTag": "ANXIOUS",
  "shortNote": "今天画得有点乱"
}
```

### 管理员流程

登录：

```http
POST /api/v1/admin/auth/login
Content-Type: application/json
```

```json
{
  "username": "admin",
  "password": "admin123456"
}
```

管理员接口也带登录返回的 token：

```http
Authorization: Bearer <admin-token>
```

常用接口：

```text
GET  /api/v1/admin/risk-events
GET  /api/v1/admin/content-rules
GET  /api/v1/admin/risk-rules
GET  /api/v1/admin/support-resources
```

## 4. AI 联调步骤

第一轮前端联调不等真实 AI。后端默认配置是：

```text
TREEHOLE_AI_PROVIDER=mock
```

mock 模式下，发送消息接口会直接返回 AI 回复，前端可以正常做聊天气泡、转念建议、追问和星空图展示。

真实 AI 或 RAG 服务准备好后，AI 同学给后端这 4 个信息：

```text
1. 服务地址，例如 http://127.0.0.1:9000/ai/v1/treehole/chat
2. 是否需要鉴权，例如 X-Api-Key
3. 超时时间建议，例如 8 秒
4. 一组真实 request/response 示例
```

当前后端已经有 `mock` 和百炼兼容客户端。独立 RAG HTTP 服务接入时，后端再按 AI 同学提供的地址补客户端，不影响第一轮前端联调。

## 5. 手动验收清单

后端 Swagger 至少点通：

```text
POST /api/v1/visitors/session
POST /api/v1/chat/sessions
POST /api/v1/chat/sessions/{sessionId}/messages
GET  /api/v1/chat/stats/star-map
POST /api/v1/admin/auth/login
```

前端浏览器 Network 至少确认：

- 请求地址是后端地址，不是前端自己的 `localhost`。
- 受保护接口都有 `Authorization: Bearer <token>`。
- 成功响应里 `code === 0`。
- 发送消息后能拿到 `assistantMessage.content`。
- 星空图接口能拿到 `points`。

AI 至少确认：

- mock 模式下前端能收到回复。
- 真实 AI 未完成时，不影响用户主流程。
- 真实 AI 接入前，返回 JSON 字段固定。

## 6. 常见问题

### 前端调用 `localhost` 不通

`localhost` 永远指向当前电脑。前端同学在自己的电脑运行页面时，`localhost:8080` 不是后端电脑。改用：

```text
http://<后端电脑局域网IP>:8080
```

### 返回未登录或无权限

检查请求头：

```http
Authorization: Bearer <token>
```

匿名身份接口和管理员登录接口不用 token，其他接口基本都要带。

### CORS 报错

后端已经对 `/api/**` 放开跨域。优先检查前端请求地址是否写错、后端是否启动、接口路径是否以 `/api/v1` 开头。

### MySQL 连接失败

确认 MySQL 已启动，数据库账号密码正确。root 有密码时用：

```powershell
.\scripts\run-dev.ps1 -DbPassword "你的MySQL密码"
```

### 默认管理员账号不安全

本地演示可以用：

```text
admin / admin123456
```

公网部署前必须换掉默认密码或禁用默认账号。

### GitHub 和公网部署不是一回事

GitHub 是放代码和文档，方便协作。公网部署是让后端服务真正运行在别人能访问的地址上。第一轮先用局域网联调，公网部署后置。
