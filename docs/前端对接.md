# 数字树洞前端对接说明

这是一份给前端同学看的后端接口交接文档。当前后端是第一版接口，已经支持匿名身份、聊天会话、文本消息、涂鸦上传、情感星空图和管理员登录。

## 1. 后端地址怎么填

如果前端和后端在同一台电脑运行：

```text
http://localhost:8080
```

如果前端同学在另一台电脑上，`localhost` 不能用，因为 `localhost` 指的是他自己的电脑。需要你在自己电脑启动后端，然后把你电脑的局域网 IP 发给他，例如：

```text
http://<后端电脑局域网IP>:8080
```

前端代码里可以先把后端地址写成一个变量：

```js
const API_BASE_URL = 'http://localhost:8080'
```

联调时再改成真实后端地址。

## 2. 统一请求规则

除了创建匿名身份和管理员登录，其他接口都要带 token：

```http
Authorization: Bearer <token>
```

后端统一返回格式：

```json
{
  "code": 0,
  "message": "ok",
  "data": {},
  "timestamp": "2026-05-05T19:00:00"
}
```

前端判断是否成功时，优先看：

```js
response.code === 0
```

## 3. 用户侧主流程

### 第一步：创建匿名身份

```http
POST /api/v1/visitors/session
```

请求体：无

返回示例：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "visitorId": 1,
    "visitorCode": "b7b7c2b6-xxxx-xxxx",
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "expiresInDays": 7
  },
  "timestamp": "2026-05-05T19:00:00"
}
```

前端要做的事：

```js
localStorage.setItem('treehole_token', data.token)
```

后续用户侧请求都带：

```http
Authorization: Bearer <localStorage 里的 token>
```

### 第二步：创建聊天会话

```http
POST /api/v1/chat/sessions
```

请求头：

```http
Authorization: Bearer <token>
Content-Type: application/json
```

请求体：

```json
{
  "companionStyle": "PEER",
  "title": "考试压力"
}
```

`companionStyle` 可选值：

```text
GENTLE_ELDER
PEER
HUMOROUS
```

返回里的 `data.id` 就是 `sessionId`。

### 第三步：发送文本消息

```http
POST /api/v1/chat/sessions/{sessionId}/messages
```

请求体：

```json
{
  "inputType": "TEXT",
  "content": "我最近因为考试很焦虑，但还是想试着说出来"
}
```

返回重点字段：

```json
{
  "data": {
    "userMessage": {},
    "assistantMessage": {
      "content": "AI 回复内容",
      "emotionTag": "ANXIOUS"
    },
    "reframeText": "转念一想...",
    "followUpQuestion": "如果愿意的话，可以再说一点...",
    "riskLevel": "NONE",
    "supportResources": []
  }
}
```

前端展示建议：

- 用户消息展示 `userMessage.content`
- AI 回复展示 `assistantMessage.content`
- 如果 `supportResources` 不为空，就展示心理援助联系方式

### 第四步：查询会话列表

```http
GET /api/v1/chat/sessions?pageNo=1&pageSize=10
```

### 第五步：查询消息列表

```http
GET /api/v1/chat/sessions/{sessionId}/messages?pageNo=1&pageSize=20
```

## 4. 涂鸦输入流程

涂鸦是两步：

### 第一步：上传 PNG 涂鸦

```http
POST /api/v1/doodles
```

请求类型：

```text
multipart/form-data
```

字段名：

```text
file
```

限制：

```text
只支持 PNG
最大 2MB
最大分辨率 1024x1024
```

返回示例：

```json
{
  "data": {
    "doodleAssetId": 1,
    "publicUrl": "http://localhost:8080/assets/doodles/xxx.png",
    "fileSize": 12345,
    "width": 800,
    "height": 800,
    "status": "UPLOADED"
  }
}
```

### 第二步：发送涂鸦消息

```http
POST /api/v1/chat/sessions/{sessionId}/messages
```

请求体：

```json
{
  "inputType": "DOODLE",
  "doodleAssetId": 1,
  "emotionTag": "ANXIOUS",
  "shortNote": "这张图像一团乱线，感觉最近脑子很乱"
}
```

`emotionTag` 可选值：

```text
CALM
SAD
ANXIOUS
ANGRY
HOPEFUL
```

## 5. 情感星空图

```http
GET /api/v1/chat/stats/star-map?days=7
```

返回示例：

```json
{
  "data": {
    "periodStart": "2026-04-29",
    "periodEnd": "2026-05-05",
    "dominantEmotion": "ANXIOUS",
    "summaryText": "最近 7 天里，焦虑是出现最多的情绪...",
    "points": [
      {
        "x": 23.4,
        "y": 80.1,
        "radius": 4,
        "color": "#F6C177",
        "emotionTag": "ANXIOUS",
        "sourceDate": "2026-05-05",
        "intensity": 0.67,
        "label": "焦虑"
      }
    ]
  }
}
```

前端渲染建议：

- 用 `x` 和 `y` 定位星星
- 用 `radius` 控制星星大小
- 用 `color` 控制星星颜色
- 用 `summaryText` 做星空图旁边的本周情绪摘要

## 6. 管理员登录

```http
POST /api/v1/admin/auth/login
```

请求体：

```json
{
  "username": "admin",
  "password": "admin123456"
}
```

返回 `token` 后，管理员接口也带：

```http
Authorization: Bearer <admin token>
```

管理员接口：

```text
GET  /api/v1/admin/risk-events
GET  /api/v1/admin/content-rules
POST /api/v1/admin/content-rules
PUT  /api/v1/admin/content-rules/{id}
GET  /api/v1/admin/risk-rules
POST /api/v1/admin/risk-rules
PUT  /api/v1/admin/risk-rules/{id}
GET  /api/v1/admin/support-resources
POST /api/v1/admin/support-resources
PUT  /api/v1/admin/support-resources/{id}
```

## 7. 现在前端可以先做什么

前端可以先开始做这些页面：

```text
匿名进入页
树洞聊天页
涂鸦画板
历史会话列表
情感星空图
管理员登录页
风险事件列表页
规则维护页
```

如果后端暂时没部署到公网，前端可以先按这份文档写接口封装和页面结构，等后端地址确定后再联调。
